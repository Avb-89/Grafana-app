# Grafana.app

**Grafana.app** — это переносная macOS-программа-«бутылка» для Grafana.

Идея простая: всё, что нужно для локального мониторинга, лежит внутри одного `.app`:

- Grafana
- Prometheus
- база Grafana
- данные Prometheus
- пользовательские скрипты
- импортируемые метрики
- логи
- служебные файлы

## Зачем это нужно

Обычная Grafana удобная, но её окружение обычно размазано по системе: конфиги отдельно, база отдельно, Prometheus отдельно, скрипты отдельно, логи отдельно.

`Grafana.app` собирает всё в одном месте:

```text
Grafana.app/
└── Contents/
    ├── Workspace/      # Grafana, Prometheus, база, логи, данные
    ├── Monitoring/     # метрики, история импорта, состояние
    └── Scripts/        # пользовательские скрипты
```

Так Grafana становится переносной: приложение можно перенести на другой Mac вместе с его базой, настройками, скриптами и накопленными метриками.

## Что умеет

- Запускает локальную Grafana.
- Запускает локальный Prometheus.
- Хранит данные внутри самого `.app`.
- Устанавливает компоненты Grafana и Prometheus вручную через раздел **Инструменты**.
- Показывает статус сервисов.
- Открывает Grafana в отдельном окне.
- Позволяет запускать пользовательские shell-скрипты.
- Позволяет запускать разные скрипты параллельно.
- Не допускает повторный запуск одного и того же скрипта, пока предыдущий экземпляр ещё работает.
- Позволяет запускать скрипты по расписанию.
- Хранит логи запуска скриптов.
- Импортирует `.openmetrics`-файлы в Prometheus.
- Хранит историю служебного состояния импорта.
- Позволяет очищать накопленные данные, чтобы приложение не разрасталось бесконтрольно.
- Имеет ограничения для контроля размера данных Prometheus.
- Не требует ручной установки Grafana и Prometheus в систему.

## Важная особенность

Это не автообновлятор.

Раздел **Инструменты** устанавливает версии компонентов, зафиксированные в приложении. Кнопка **Проверка доступа к репозиторию** не ищет latest-версии в интернете, а проверяет доступность нужных архивов и показывает план установки.

Кнопка **Установить компоненты** скачивает и устанавливает Grafana и Prometheus внутрь:

```text
Grafana.app/Contents/Workspace
```

## Размер приложения

До установки компонентов:

```text
около 2 МБ
```

После установки Grafana и Prometheus:

```text
примерно 2 ГБ
```

Дальше размер может расти из-за:

- базы Grafana;
- истории Prometheus;
- логов;
- временных файлов установки;
- пользовательских скриптов;
- накопленных метрик.

Для этого в приложении есть очистка данных и служебных папок.

## Очистка

Приложение умеет очищать накопленные данные внутри Workspace:

- историю Prometheus;
- базу Grafana;
- логи;
- плагины;
- временные файлы;
- кэш установки;
- staging-папки;
- backup-папки.

При этом пользовательские папки:

```text
Contents/Scripts
Contents/Monitoring
```
не удаляются для удаление скриптов надо зайти в папку и удалять вручную.

## Как работает импорт метрик

Пользовательские скрипты могут складывать `.openmetrics`-файлы в:

```text
Grafana.app/Contents/Monitoring/metrics/history/pending
```

Приложение периодически проверяет эту папку и импортирует файлы в Prometheus через `promtool`.

После успешного импорта файл удаляется из `pending`, а информация об импорте записывается в служебный registry.

## Пример первого запуска

1. Запустить приложение.
2. Открыть раздел **Инструменты**.
3. Нажать **Проверка доступа к репозиторию**.
4. Нажать **Установить компоненты**.
5. Дождаться установки Grafana и Prometheus.
6. Нажать **Запустить Grafana**.
7. Открыть Grafana.
8. Добавить datasource Prometheus:
   
```text
URL: http://127.0.0.1:9090
```

9. Создавать dashboards как в обычной Grafana.

## Где лежат данные

Основная рабочая папка:

```text
Grafana.app/Contents/Workspace
```

Примерная структура:

```text
Workspace/
├── grafana/
│   ├── bin/
│   ├── conf/
│   ├── data/
│   ├── logs/
│   ├── plugins/
│   └── provisioning/
├── prometheus/
│   ├── prometheus
│   ├── promtool
│   ├── prometheus.yml
│   └── data/
├── logs/
├── updates/
│   ├── downloads/
│   ├── staging/
│   └── logs/
└── backups/
```

Метрики и скрипты:

```text
Contents/
├── Monitoring/
│   ├── metrics/
│   └── state/
└── Scripts/
```

## Сборка

Проект собирается в Xcode как обычное macOS SwiftUI-приложение.

Компоненты Grafana и Prometheus не входят в репозиторий и устанавливаются уже из самого приложения.

## Статус проекта

Проект экспериментальный.

Основная цель — сделать удобную переносную Grafana-бутылку для мониторинга, тестов, скриптов и небольших автономных стендов. Особенно удобно для мониторинга локальных контейнеров проекта, при разработке.
